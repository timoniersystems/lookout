## Quick Start

### 1. Start All Services

```bash
# Generate TLS certificates for nginx
./scripts/generate-certs.sh

# Start all services
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f
```

### 2. Access the Application

- **Web UI (HTTPS):** https://localhost:7443
- **Web UI (HTTP):** http://localhost:7080 (redirects to HTTPS)
- **Dgraph Ratel UI:** http://localhost:8000
- **Dgraph HTTP API:** http://localhost:8080
- **Dgraph gRPC API:** localhost:9080

**Note:** Browser will show security warning for self-signed certificates - this is expected in development.

### 3. Stop Services

```bash
# Stop services
docker-compose down

# Stop and remove volumes (deletes all data)
docker-compose down -v
```

## Services Overview

### lookout (Web Application)

The main Lookout web UI application.

**Container:** `lookout-app`
**Image:** Built from Dockerfile
**Ports:** 3000 (internal only, accessed via nginx)
**Dependencies:** dgraph-alpha

**Features:**
- CVE analysis via web interface
- SBOM upload and processing
- Dependency path visualization
- Built with Go + Echo framework
- Multi-architecture support (AMD64, ARM64)

### nginx (Reverse Proxy)

TLS termination and reverse proxy for the web application.

**Container:** `lookout-nginx`
**Image:** `nginx:1.25-alpine`
**Ports:**
- 7080 → 80 (HTTP, redirects to HTTPS)
- 7443 → 443 (HTTPS)

**Features:**
- TLS 1.2/1.3 only
- HTTP/2 support
- Security headers (HSTS, CSP, X-Frame-Options)
- Rate limiting on uploads (10 req/s)
- Gzip compression
- Static file caching

**Configuration:**
- Main config: `nginx/nginx.conf`
- Virtual host: `nginx/conf.d/lookout.conf`
- Certificates: `nginx/certs/` (generated by `scripts/generate-certs.sh`)

See [TLS Setup Guide](TLS_SETUP.md) for details.

### dgraph-zero (Coordinator)

Dgraph cluster coordinator and metadata server.

**Container:** `lookout-dgraph-zero`
**Image:** `dgraph/dgraph:v24.0.0`
**Ports:**
- 5080 (cluster communication)
- 6080 (HTTP API)

**Features:**
- Cluster coordination
- Timestamp oracle
- Group assignments

### dgraph-alpha (Data Server)

Dgraph graph database server for dependency graph storage.

**Container:** `lookout-dgraph-alpha`
**Image:** `dgraph/dgraph:v24.0.0`
**Ports:**
- 8080 (HTTP API)
- 9080 (gRPC API)

**Features:**
- Graph data storage
- GraphQL API
- Dependency path queries
- Transactional operations

### dgraph-ratel (UI)

Web-based Dgraph data explorer.

**Container:** `lookout-dgraph-ratel`
**Image:** `dgraph/ratel:v21.03.0`
**Ports:** 8000

**Features:**
- Visual graph exploration
- Query editor
- Schema viewer
- Mutation builder

### dgraph (Standalone)

Single-container Dgraph instance for testing (profile: standalone).

**Container:** `lookout-dgraph-standalone`
**Profile:** `standalone`
**Ports:** 5080, 6080, 8080

**Usage:**
```bash
docker-compose --profile standalone up -d dgraph
```

Use this for CLI-only workflows without the web UI.

## Port Mappings

| Service | Container Port | Host Port | Protocol | Description |
|---------|---------------|-----------|----------|-------------|
| nginx | 80 | 7080 | HTTP | HTTP (redirects to HTTPS) |
| nginx | 443 | 7443 | HTTPS | Main web UI (TLS) |
| lookout | 3000 | - | HTTP | Internal only (via nginx) |
| dgraph-alpha | 8080 | 8080 | HTTP | Dgraph HTTP API |
| dgraph-alpha | 9080 | 9080 | gRPC | Dgraph gRPC API |
| dgraph-zero | 5080 | 5080 | TCP | Cluster communication |
| dgraph-zero | 6080 | 6080 | HTTP | Zero HTTP API |
| dgraph-ratel | 8000 | 8000 | HTTP | Dgraph UI |

**Why Port 7080/7443?**
- Ports 80/443 require root privileges on most systems
- Port 8080 already used by dgraph-alpha
- Port 7080/7443 are non-privileged and available

## Service Profiles

Docker Compose profiles allow running subsets of services.

### Default Profile

Runs the full web application stack.

```bash
docker-compose up -d
```

**Includes:**
- lookout (web UI)
- nginx (reverse proxy)
- dgraph-zero
- dgraph-alpha
- dgraph-ratel

### Standalone Profile

Runs only Dgraph for CLI usage.

```bash
docker-compose --profile standalone up -d
```

**Includes:**
- dgraph (single container with zero + alpha)

**Use case:** CLI-only workflows without web UI

```bash
# Example CLI usage with standalone Dgraph
export DGRAPH_HOST=localhost
export DGRAPH_PORT=9080
lookout -sbom examples/cyclonedx-sbom-example.json \
  -dep-path 'pkg:composer/asm89/stack-cors@1.3.0'
```

## Environment Variables

### Application Configuration

Create a `.env` file in the project root:

```bash
# Dgraph connection
DGRAPH_HOST=dgraph-alpha
DGRAPH_PORT=9080

# Web server
SERVER_PORT=3000

# Build arguments (optional)
GO_VERSION=1.23.5
ALPINE_VERSION=3.21
TRIVY_VERSION=0.58.2
```

### Service-Specific Variables

**lookout service:**
- `DGRAPH_HOST` - Dgraph hostname (default: `dgraph-alpha`)
- `DGRAPH_PORT` - Dgraph gRPC port (default: `9080`)
- `SERVER_PORT` - Internal HTTP port (default: `3000`)
- `NVD_API_KEY` - NVD API key for faster CVE lookups (optional)

**Build arguments:**
- `GO_VERSION` - Go version for builder stage
- `ALPINE_VERSION` - Alpine Linux version
- `TRIVY_VERSION` - Trivy scanner version

## Common Commands

### Service Management

```bash
# Start all services
docker-compose up -d

# Start specific service
docker-compose up -d lookout

# Stop all services
docker-compose down

# Restart a service
docker-compose restart lookout

# View service status
docker-compose ps

# View resource usage
docker stats
```

### Logs and Debugging

```bash
# View all logs
docker-compose logs

# Follow logs in real-time
docker-compose logs -f

# View logs for specific service
docker-compose logs lookout
docker-compose logs nginx

# Last 50 lines
docker-compose logs --tail=50

# Logs since timestamp
docker-compose logs --since 2024-01-01T10:00:00
```

### Container Access

```bash
# Execute command in running container
docker exec lookout-app curl http://localhost:3000/health

# Interactive shell
docker exec -it lookout-app sh

# View container processes
docker exec lookout-app ps aux

# Check binary
docker exec lookout-app ls -lh /app/lookout-ui
```

### Data Management

```bash
# Backup Dgraph data
docker exec lookout-dgraph-alpha \
  curl -X POST localhost:8080/admin/export

# Drop all data (destructive!)
docker exec lookout-dgraph-alpha \
  curl -X POST localhost:8080/alter -d '{"drop_all": true}'

# Remove volumes (deletes all data)
docker-compose down -v
```

### Rebuild and Update

```bash
# Rebuild image after code changes
docker-compose build lookout

# Rebuild without cache
docker-compose build --no-cache lookout

# Pull latest base images
docker-compose pull

# Recreate containers
docker-compose up -d --force-recreate
```

## Health Checks

All services have health checks configured.

### Check Health Status

```bash
# View health status
docker-compose ps

# Expected output:
# lookout-nginx          Up (healthy)
# lookout-app            Up (healthy)
# lookout-dgraph-alpha   Up (healthy)
# lookout-dgraph-zero    Up (healthy)
```

### Manual Health Checks

```bash
# Test lookout UI (internal)
docker exec lookout-app curl -f http://localhost:3000/health

# Test nginx HTTPS
curl -k https://localhost:7443/health

# Test nginx HTTP redirect
curl -I http://localhost:7080/

# Test dgraph-alpha
curl http://localhost:8080/health

# Test dgraph-zero
curl http://localhost:6080/health
```

### Health Check Configuration

Services wait for dependencies to be healthy before starting:

```yaml
lookout:
  depends_on:
    dgraph-alpha:
      condition: service_healthy

dgraph-alpha:
  depends_on:
    dgraph-zero:
      condition: service_healthy
```

**Startup sequence:**
1. dgraph-zero starts and becomes healthy (20s)
2. dgraph-alpha starts and becomes healthy (20s)
3. lookout starts and becomes healthy (5s)
4. nginx starts and becomes healthy (10s)

**Total startup time:** ~55 seconds

## Troubleshooting

### Services Unhealthy

**Check logs:**
```bash
docker-compose logs lookout
docker-compose logs nginx
docker-compose logs dgraph-alpha
```

**Common issues:**
- Dgraph not healthy: Wait 30s for initialization
- Nginx unhealthy: Check certificates exist in `nginx/certs/`
- Lookout unhealthy: Check dgraph-alpha is healthy first

### Port Conflicts

**Error:** `Bind for 0.0.0.0:7080 failed: port is already allocated`

**Solution:**
```bash
# Find process using port
lsof -i :7080

# Change port in docker-compose.yml
ports:
  - "9080:80"  # Use different host port
```

### Connection Refused

**Symptom:** Can't access https://localhost:7443

**Checks:**
```bash
# Is nginx running?
docker ps | grep nginx

# Are certificates generated?
ls -la nginx/certs/

# Check nginx logs
docker logs lookout-nginx

# Test internal connectivity
docker exec lookout-nginx curl http://lookout:3000/health
```

### 502 Bad Gateway

**Cause:** lookout-app not responding

**Checks:**
```bash
# Is lookout running?
docker ps | grep lookout-app

# Check lookout logs
docker logs lookout-app

# Test direct connection
docker exec lookout-app curl http://localhost:3000/health

# Check network
docker network inspect lookout_lookout-net | grep lookout
```

### Certificate Errors

**Symptom:** "Your connection is not private" in browser

**Expected behavior:** This is normal with self-signed certificates

**Solutions:**
1. Development: Click "Advanced" → "Proceed to localhost"
2. Production: Use Let's Encrypt (see [TLS Setup Guide](TLS_SETUP.md))

### Dgraph Connection Issues

**Symptom:** Lookout can't connect to Dgraph

**Checks:**
```bash
# Is dgraph-alpha healthy?
docker ps | grep dgraph-alpha

# Test connection from lookout container
docker exec lookout-app nc -zv dgraph-alpha 9080

# Check environment variables
docker exec lookout-app env | grep DGRAPH
```

### Slow Performance

**Check resource usage:**
```bash
docker stats

# If memory/CPU high:
docker-compose restart lookout
docker system prune -a  # Free up disk space
```

## Production Deployment

### Pre-Production Checklist

- [ ] Generate real TLS certificates (Let's Encrypt)
- [ ] Configure external secrets management
- [ ] Set up volume backups
- [ ] Configure log aggregation
- [ ] Set resource limits
- [ ] Enable monitoring/alerting
- [ ] Configure firewall rules
- [ ] Set up reverse proxy (if not using nginx)
- [ ] Configure domain name and DNS

### Production docker-compose.yml Changes

```yaml
# Add resource limits
services:
  lookout:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G

  # Persist Dgraph data
  dgraph-alpha:
    volumes:
      - dgraph-data:/dgraph

volumes:
  dgraph-data:
    driver: local
```

### Let's Encrypt Certificates

See [TLS Setup Guide](TLS_SETUP.md#production-lets-encrypt) for automated certificate setup.

Quick version:
```bash
# Install certbot
sudo apt-get install certbot

# Get certificate
sudo certbot certonly --standalone \
  -d yourdomain.com \
  --email you@yourdomain.com \
  --agree-tos

# Copy to nginx
sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem nginx/certs/cert.pem
sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem nginx/certs/key.pem

# Update nginx config
sed -i 's/server_name localhost;/server_name yourdomain.com;/' nginx/conf.d/lookout.conf

# Reload nginx
docker-compose restart nginx
```

### Monitoring

**Recommended stack:**
- Prometheus + Grafana for metrics
- ELK or Loki for log aggregation
- Uptime monitoring (UptimeRobot, Pingdom)

**Health check endpoints:**
- `https://yourdomain.com/health` - Main health check
- `https://yourdomain.com/ready` - Readiness probe
- `http://dgraph-alpha:8080/health` - Dgraph health

### Backup Strategy

**Dgraph data:**
```bash
# Automated backup cron job
0 2 * * * docker exec lookout-dgraph-alpha curl -X POST localhost:8080/admin/export
```

**Certificates:**
```bash
# Backup nginx certs
tar -czf nginx-certs-backup.tar.gz nginx/certs/
```

**GCP:**
- Cloud Run for containers
- Cloud Load Balancing + managed certs
- Firestore/Datastore

**Azure:**
- Container Instances
- Application Gateway + Key Vault certs
- Cosmos DB
