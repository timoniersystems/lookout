## Quick Start

### 1. Start All Services

```bash
# Generate TLS certificates for nginx
./scripts/generate-certs.sh

# Start all services
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f
```

### 2. Access the Application

- **Web UI (HTTPS):** https://localhost:7443
- **Web UI (HTTP):** http://localhost:7080 (redirects to HTTPS)
- **Dgraph Ratel UI:** http://localhost:8000
- **Dgraph HTTP API:** http://localhost:8080
- **Dgraph gRPC API:** localhost:9080

**Note:** Browser will show security warning for self-signed certificates - this is expected in development.

### 3. Stop Services

```bash
# Stop services
docker-compose down

# Stop and remove volumes (deletes all data)
docker-compose down -v
```

## Services Overview

### lookout (Web Application)

The main Lookout web UI application.

**Container:** `lookout-app`
**Image:** Built from Dockerfile
**Ports:** 3000 (internal only, accessed via nginx)
**Dependencies:** dgraph-alpha

**Features:**
- CVE analysis via web interface
- SBOM upload and processing
- Dependency path visualization
- Built with Go + Echo framework
- Distroless base image for security (gcr.io/distroless/static-debian12)
- Multi-architecture support (AMD64, ARM64)

### nginx (Reverse Proxy)

TLS termination and reverse proxy for the web application.

**Container:** `lookout-nginx`
**Image:** `nginx:1.25-alpine`
**Ports:**
- 7080 → 80 (HTTP, redirects to HTTPS)
- 7443 → 443 (HTTPS)

**Features:**
- TLS 1.2/1.3 only
- HTTP/2 support
- Security headers (HSTS, CSP, X-Frame-Options)
- Rate limiting on uploads (10 req/s)
- Gzip compression
- Static file caching

**Configuration:**
- Main config: `nginx/nginx.conf`
- Virtual host: `nginx/conf.d/lookout.conf`
- Certificates: `nginx/certs/` (generated by `scripts/generate-certs.sh`)

See [TLS/HTTPS Setup](#tlshttps-setup) section below for details.

### dgraph-zero (Coordinator)

Dgraph cluster coordinator and metadata server.

**Container:** `lookout-dgraph-zero`
**Image:** `dgraph/dgraph:v24.0.0`
**Ports:**
- 5080 (cluster communication)
- 6080 (HTTP API)

**Features:**
- Cluster coordination
- Timestamp oracle
- Group assignments

### dgraph-alpha (Data Server)

Dgraph graph database server for dependency graph storage.

**Container:** `lookout-dgraph-alpha`
**Image:** `dgraph/dgraph:v24.0.0`
**Ports:**
- 8080 (HTTP API)
- 9080 (gRPC API)

**Features:**
- Graph data storage
- GraphQL API
- Dependency path queries
- Transactional operations

### dgraph-ratel (UI)

Web-based Dgraph data explorer.

**Container:** `lookout-dgraph-ratel`
**Image:** `dgraph/ratel:v21.03.0`
**Ports:** 8000

**Features:**
- Visual graph exploration
- Query editor
- Schema viewer
- Mutation builder

### dgraph (Standalone)

Single-container Dgraph instance for testing (profile: standalone).

**Container:** `lookout-dgraph-standalone`
**Profile:** `standalone`
**Ports:** 5080, 6080, 8080

**Usage:**
```bash
docker-compose --profile standalone up -d dgraph
```

Use this for CLI-only workflows without the web UI.

## Port Mappings

| Service | Container Port | Host Port | Protocol | Description |
|---------|---------------|-----------|----------|-------------|
| nginx | 80 | 7080 | HTTP | HTTP (redirects to HTTPS) |
| nginx | 443 | 7443 | HTTPS | Main web UI (TLS) |
| lookout | 3000 | - | HTTP | Internal only (via nginx) |
| dgraph-alpha | 8080 | 8080 | HTTP | Dgraph HTTP API |
| dgraph-alpha | 9080 | 9080 | gRPC | Dgraph gRPC API |
| dgraph-zero | 5080 | 5080 | TCP | Cluster communication |
| dgraph-zero | 6080 | 6080 | HTTP | Zero HTTP API |
| dgraph-ratel | 8000 | 8000 | HTTP | Dgraph UI |

**Why Port 7080/7443?**
- Ports 80/443 require root privileges on most systems
- Port 8080 already used by dgraph-alpha
- Port 7080/7443 are non-privileged and available

## Service Profiles

Docker Compose profiles allow running subsets of services.

### Default Profile

Runs the full web application stack.

```bash
docker-compose up -d
```

**Includes:**
- lookout (web UI)
- nginx (reverse proxy)
- dgraph-zero
- dgraph-alpha
- dgraph-ratel

### Standalone Profile

Runs only Dgraph for CLI usage.

```bash
docker-compose --profile standalone up -d
```

**Includes:**
- dgraph (single container with zero + alpha)

**Use case:** CLI-only workflows without web UI

```bash
# Example CLI usage with standalone Dgraph
export DGRAPH_HOST=localhost
export DGRAPH_PORT=9080
lookout -sbom examples/cyclonedx-sbom-example.json \
  -dep-path 'pkg:composer/asm89/stack-cors@1.3.0'
```

## Environment Variables

### Application Configuration

Create a `.env` file in the project root:

```bash
# Dgraph connection
DGRAPH_HOST=dgraph-alpha
DGRAPH_PORT=9080

# Web server
SERVER_PORT=3000

# Build arguments (optional)
GO_VERSION=1.26.0
TRIVY_VERSION=0.69.1
# Note: Runtime uses distroless/static-debian12 for security
```

### Service-Specific Variables

**lookout service:**
- `DGRAPH_HOST` - Dgraph hostname (default: `dgraph-alpha`)
- `DGRAPH_PORT` - Dgraph gRPC port (default: `9080`)
- `SERVER_PORT` - Internal HTTP port (default: `3000`)
- `NVD_API_KEY` - NVD API key for faster CVE lookups (optional)

**Build arguments:**
- `GO_VERSION` - Go version for builder stage (default: 1.26.0)
- `TRIVY_VERSION` - Trivy scanner version (default: 0.69.1)
- Runtime uses distroless/static-debian12:nonroot for minimal attack surface

```bash
# Test dgraph-alpha
curl http://localhost:8080/health

# Test dgraph-zero
curl http://localhost:6080/health
```

### Health Check Configuration

Services wait for dependencies to be healthy before starting:

```yaml
lookout:
  depends_on:
    dgraph-alpha:
      condition: service_healthy

dgraph-alpha:
  depends_on:
    dgraph-zero:
      condition: service_healthy
```

**Startup sequence:**
1. dgraph-zero starts and becomes healthy (20s)
2. dgraph-alpha starts and becomes healthy (20s)
3. lookout starts and becomes healthy (5s)
4. nginx starts and becomes healthy (10s)

**Total startup time:** ~55 seconds

## TLS/HTTPS Setup

### Overview

Lookout uses nginx as a reverse proxy for TLS termination, providing secure HTTPS access to the web application.

**Architecture:**
```
Internet → Nginx (TLS Termination) → Lookout UI (HTTP)
           ↓
        Port 443/HTTPS              Port 3000/HTTP
        (Public)                    (Internal only)
```

**Benefits:**
- ✅ Separation of concerns (TLS vs application logic)
- ✅ Better performance (nginx optimized for TLS)
- ✅ Easier certificate management
- ✅ Additional features (rate limiting, caching, load balancing)
- ✅ Security headers automatically added
- ✅ HTTP → HTTPS redirect

### Development Setup (Self-Signed Certificates)

For local development, use self-signed certificates:

```bash
# Generate development certificates
./scripts/generate-certs.sh

# Start all services
docker-compose up -d

# Access via HTTPS
open https://localhost:7443

# Or via HTTP (redirects to HTTPS)
open http://localhost:7080
```

**Note:** Browser will show security warning for self-signed certificates - this is expected in development.

### Production Setup (Let's Encrypt)

For production deployments with real domain names:

```bash
# Install certbot
sudo apt-get install certbot python3-certbot-nginx

# Get certificate (stop nginx first if running)
sudo certbot certonly --standalone \
  -d yourdomain.com \
  -d www.yourdomain.com \
  --email you@yourdomain.com \
  --agree-tos

# Copy certificates to nginx directory
sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem nginx/certs/cert.pem
sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem nginx/certs/key.pem

# Update nginx config with your domain
sed -i 's/server_name localhost;/server_name yourdomain.com www.yourdomain.com;/' nginx/conf.d/lookout.conf

# Restart nginx
docker-compose restart nginx
```

### Security Features

**TLS Configuration:**
- ✅ TLS 1.2 & 1.3 only (older versions disabled)
- ✅ Strong cipher suites (ECDHE-based, forward secrecy)
- ✅ OCSP stapling (faster certificate validation)
- ✅ Session caching (improves TLS handshake performance)

**Security Headers:**

All responses include:
- **HSTS** - Force HTTPS for 2 years + preload
- **X-Frame-Options** - Prevent clickjacking
- **X-Content-Type-Options** - Prevent MIME sniffing
- **X-XSS-Protection** - XSS filter
- **Referrer-Policy** - Control referrer information
- **CSP** - Content Security Policy

**Rate Limiting:**
- Upload endpoints: 10 requests/second per IP
- Burst: Up to 5 requests
- Status: Returns 429 (Too Many Requests)

### Performance Features

✅ **HTTP/2** - Multiplexing, header compression
✅ **Gzip compression** - Reduces bandwidth by ~70%
✅ **Connection pooling** - Keepalive to backend
✅ **Static file caching** - 1 year cache headers
✅ **No buffering** - Real-time responses

### Endpoints

| URL | Description | Features |
|-----|-------------|----------|
| `http://localhost:7080/` | Redirects to HTTPS | 301 Permanent Redirect |
| `https://localhost:7443/` | Home page | Full app access |
| `https://localhost:7443/health` | Health check | No logging, fast response |
| `https://localhost:7443/ready` | Readiness probe | Kubernetes compatible |
| `https://localhost:7443/static/*` | Static files | 1 year cache |
| `https://localhost:7443/upload` | Upload endpoint | Rate limited |

### Certificate Management

**Automated Renewal (Let's Encrypt):**

Add to crontab:
```bash
# Check for renewal twice daily
0 0,12 * * * certbot renew --quiet --post-hook "docker-compose -f /path/to/lookout/docker-compose.yml restart nginx"
```

Or use systemd timer:
```bash
sudo systemctl enable certbot-renew.timer
sudo systemctl start certbot-renew.timer
```

**Manual Renewal:**
```bash
# Renew certificates
sudo certbot renew

# Copy new certificates
sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem nginx/certs/cert.pem
sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem nginx/certs/key.pem

# Reload nginx (no downtime)
docker exec lookout-nginx nginx -s reload
```

### TLS Troubleshooting

**Certificate Errors:**
```bash
# Check certificates exist
ls -la nginx/certs/

# Verify certificate
openssl x509 -in nginx/certs/cert.pem -text -noout

# Check private key
openssl rsa -in nginx/certs/key.pem -check

# Verify cert matches key
openssl x509 -noout -modulus -in nginx/certs/cert.pem | openssl md5
openssl rsa -noout -modulus -in nginx/certs/key.pem | openssl md5
# (should match)
```

**Connection Issues:**
```bash
# Check if nginx is listening
docker exec lookout-nginx netstat -tlnp

# Verify port bindings
docker port lookout-nginx

# Test from inside nginx container
docker exec lookout-nginx wget -O- http://lookout:3000/health
```

**Performance Issues:**
```bash
# Check worker connections
docker exec lookout-nginx grep -E "worker_processes|worker_connections" /etc/nginx/nginx.conf

# Monitor active connections
docker exec lookout-nginx sh -c "while true; do netstat -an | grep :443 | wc -l; sleep 1; done"

# Check for errors
docker logs lookout-nginx | grep -i error
```

### Security Hardening

**Additional Security Headers:**

Add to `nginx/conf.d/lookout.conf`:
```nginx
# Additional security
add_header Permissions-Policy "geolocation=(), microphone=(), camera=()" always;
add_header X-Permitted-Cross-Domain-Policies "none" always;
```

**Disable Unnecessary Methods:**
```nginx
if ($request_method !~ ^(GET|HEAD|POST)$ ) {
    return 444;
}
```

**IP Whitelisting (if needed):**
```nginx
geo $allowed_ip {
    default 0;
    1.2.3.4 1;  # Allow specific IP
    10.0.0.0/8 1;  # Allow private network
}

server {
    if ($allowed_ip = 0) {
        return 403;
    }
}
```

### Alternative Reverse Proxies

**Caddy (Easiest - Auto TLS):**
```yaml
caddy:
  image: caddy:2-alpine
  ports:
    - "80:80"
    - "443:443"
  volumes:
    - ./Caddyfile:/etc/caddy/Caddyfile
    - caddy_data:/data
```

```caddyfile
yourdomain.com {
    reverse_proxy lookout:3000
    # Automatic HTTPS!
}
```

**Pros:** Automatic Let's Encrypt, simple config
**Cons:** Less battle-tested than nginx

**Traefik (Best for Microservices):**
```yaml
traefik:
  image: traefik:v2.10
  command:
    - "--providers.docker=true"
    - "--entrypoints.web.address=:80"
    - "--entrypoints.websecure.address=:443"
    - "--certificatesresolvers.le.acme.email=you@example.com"

lookout:
  labels:
    - "traefik.http.routers.lookout.rule=Host(`yourdomain.com`)"
    - "traefik.http.routers.lookout.tls.certresolver=le"
```

**Pros:** Service discovery, automatic HTTPS
**Cons:** More complex setup

## Troubleshooting

### Services Unhealthy

**Check logs:**
```bash
docker-compose logs lookout
docker-compose logs nginx
docker-compose logs dgraph-alpha
```

**Common issues:**
- Dgraph not healthy: Wait 30s for initialization
- Nginx unhealthy: Check certificates exist in `nginx/certs/`
- Lookout unhealthy: Check dgraph-alpha is healthy first

### Port Conflicts

**Error:** `Bind for 0.0.0.0:7080 failed: port is already allocated`

**Solution:**
```bash
# Find process using port
lsof -i :7080

# Change port in docker-compose.yml
ports:
  - "9080:80"  # Use different host port
```

### Connection Refused

**Symptom:** Can't access https://localhost:7443

**Checks:**
```bash
# Is nginx running?
docker ps | grep nginx

# Are certificates generated?
ls -la nginx/certs/

# Check nginx logs
docker logs lookout-nginx

# Test internal connectivity
docker exec lookout-nginx curl http://lookout:3000/health
```

### 502 Bad Gateway

**Cause:** lookout-app not responding

**Checks:**
```bash
# Is lookout running?
docker ps | grep lookout-app

# Check lookout logs
docker logs lookout-app

# Test direct connection
docker exec lookout-app curl http://localhost:3000/health

# Check network
docker network inspect lookout_lookout-net | grep lookout
```

### Certificate Errors

**Symptom:** "Your connection is not private" in browser

**Expected behavior:** This is normal with self-signed certificates

**Solutions:**
1. Development: Click "Advanced" → "Proceed to localhost"
2. Production: Use Let's Encrypt (see [TLS/HTTPS Setup](#production-setup-lets-encrypt))

### Dgraph Connection Issues

**Symptom:** Lookout can't connect to Dgraph

**Checks:**
```bash
# Is dgraph-alpha healthy?
docker ps | grep dgraph-alpha

# Test connection from lookout container
docker exec lookout-app nc -zv dgraph-alpha 9080

# Check environment variables
docker exec lookout-app env | grep DGRAPH
```

### Slow Performance

**Check resource usage:**
```bash
docker stats

# If memory/CPU high:
docker-compose restart lookout
docker system prune -a  # Free up disk space
```

## Production Deployment

### Pre-Production Checklist

- [ ] Generate real TLS certificates (Let's Encrypt)
- [ ] Configure external secrets management
- [ ] Set up volume backups
- [ ] Configure log aggregation
- [ ] Set resource limits
- [ ] Enable monitoring/alerting
- [ ] Configure firewall rules

### Production docker-compose.yml Changes

```yaml
# Add resource limits
services:
  lookout:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G

  # Persist Dgraph data
  dgraph-alpha:
    volumes:
      - dgraph-data:/dgraph

volumes:
  dgraph-data:
    driver: local
```
